# 2026-04-16

## セッション1: Web版デプロイと音声・会話体験の一括修正

### 実装した機能
- **Proactive Audio モード**: Gemini 応答に `[mode:aizuchi|respond|silent]` タグを付与し、応答の種類を制御。silent 時は再生せず。
- **TTS 文節ストリーミング再生**: AudioContext と共有 `nextPlayTime` で複数 AudioBufferSource をシームレス連結。各 chunk に GainNode を挟んでフェード可能に。
- **フェードアウト barge-in**: `stopAllTts({ fadeMs: 150 })` で全 BufferSource の gain を ~150ms で落としてから stop。突然ブツッと切れなくなった。
- **barge-in 頭切れ対策**: RMS 連続検知前の候補フレームを `bargeInPendingFramesRef` に貯め、barge-in 確定時に Deepgram WS へ flush。「ぽこあポケモン」→「パルデアポケモン」に改善。
- **barge-in 時の履歴汚染修正**: `triggerBargeIn` で `messages` から末尾の空/極短 assistant メッセージを削除。AI が直前会話を忘れる症状を解消。
- **Live2D リップシンク復活**: 新 TTS が BufferSource 中心になり `createMediaElementSource` 経路が死んでいたので、App 側で共有 AnalyserNode を作り `Live2DAvatar` に `externalAnalyser` prop で渡す設計に変更。
- **イントネーション改善**: `isSpeakable` を文末 (。！？) 優先に戻し、句読点分割は長さ ≥15、最後の砦として長さ ≥40 に閾値を戻した。切れ目が自然に。

### 修正したバグ
- **MSAL redirect loop (localhost:37212 へリダイレクト)**: Vite ビルド時に `VITE_AZURE_REDIRECT_URI` が焼き込まれていた。`client/src/lib/msal.ts` の `MSAL_REDIRECT_URI` をブラウザ環境では `window.location.origin` を優先するよう修正。
- **COOP で popup が閉じれない**: `loginPopup` は Cross-Origin-Opener-Policy で `window.closed` が読めず詰まる。非 Electron では `loginRedirect` を使うように変更。
- **AADSTS50011 redirect mismatch**: Cloud Run の 2 種類の URL（プロジェクト番号形式と `<hash>-an.a.run.app` 形式）のうち、ハッシュ形式が Azure AD 未登録だった。人見さんに一括依頼して両方登録。

### 備考
- **Cloud Run + Azure AD の教訓を永続化**: 1 サービス = 2 URL である事実を見落とし、人見さんに段階的に追加依頼してしまった。ユーザーから強く指摘された（「人見くんに迷惑かけないでよ？」「何度も依頼すると本来の依頼が後回しになる」）。以降、外部管理者への依頼は **必要な全項目を一括で**。
  - `~/.claude/includes/azure-ad.md` に「Cloud Run デプロイ時の redirect URI 登録（超重要・失敗禁止）」セクションを追加
  - `~/.claude/skills/new-app-setup/skill.md` からも参照
  - assess-kun の memory に feedback として永続化: `feedback_external_admin_requests.md`
- **Gemini ストリーミング + タグ設計**: `[mode:xxx][表情:yyy]` を先頭で解釈し、本文を再生 queue へ投入。silent モードは reader.cancel() で中断。
- **AudioContext シームレス連結のキモ**: 各 chunk の `startTime = max(ctx.currentTime, nextPlayTime)`、`nextPlayTime = startTime + buffer.duration`。`onended` で queue 先頭を即 fetch すると TTS 生成待ちも隠せる。
