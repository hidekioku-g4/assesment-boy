## セッション2: 科学的知見に基づくプロンプト改修

### 調査した研究領域（16トピック）
- **心理学**: 傾聴(Itzchakov 2022)、自己開示(Aron 1997)、フォローアップ質問(Huang 2017)、ターンテイキング(Stivers 2009)、感情バリデーション(Linehan/Shenk 2022)
- **神経科学・行動科学**: 自己開示の報酬系(Tamir 2012)、治療的同盟(Fluckiger 2018)、動機づけ面接MI(Miller & Rollnick)、ポジティブ心理学PPI(Carr 2021)、自己決定理論SDT(Ryan 2023)、ピークエンドの法則(Cojuharenco 2022)
- **チャットボット研究**: 温かさvs有能さ(Zhang 2025)、擬人化(Kim 2024)、修復戦略(Gnewuch 2025)、応答長(JMIR 2025)、日次チェックイン効果(Fitzpatrick 2017)

### 実装した改善（6項目、すべてプロンプト修正）
1. **MI Reflection技法**: 返し方の原則を「受け止める」→「相手の言葉を反映する（言い換えて返す）」に強化
2. **SDT 自律性支援言語**: 「言葉選びの原則」セクション追加。統制的表現を排除し、自律性を尊重する言い回しに
3. **治療的同盟の目標共有**: openingフェーズに「今日はどんな一日にしたいですか？」で方向性共有
4. **PPI 強み発見**: deepeningに「相手の良いところに気づいたら伝える」セクション追加（strength-spotting）
5. **ピークエンドの法則**: セッション終了を「最重要」に格上げ。具体的内容への言及+肯定的締め+次回期待の3要素必須に
6. **自己開示の相互性**: 会話例を「反映→自分事化」の循環パターンに。NGパターンに安易な励まし・押しつけ表現を追加

### アセスメント質問の自律性支援化
- 「昨日ちゃんと寝れました？」→「今朝の目覚めはどんな感じでした？」
- 「朝起きれてるか」→「最近のペースはどうですか？」
- 判断を含まない、オープンな聞き方に統一

## セッション3: バグ修正 + 呼び名機能 + 自動保存

### 修正したバグ
- **レスポンス遅延（3-9秒）**: Gemini 2.5-flash の thinking が原因。`thinkingBudget: 0` で ~1秒に短縮
- **会話履歴消失（history:0）**: `sendChatMessage` の useCallback が `messages` state をクロージャで古い値で参照していた。`chatMessagesRef = useRef()` パターンで解決
- **口パク開きっぱなし**: TTS停止後に Live2D `ParamMouthOpenY` がリセットされない問題。`isSpeaking=false` 時に 0 にリセットする else 分岐追加
- **会話が支離滅裂になる**: history:0 バグの副次的影響。履歴修正で解決

### 実装した機能
- **呼び名（preferredName）機能**: 初回会話で「なんて呼びましょうか？」と尋ね、BQ user_profiles.notes に `呼び方: ○○さん` を保存。以降のセッションで利用。session-summary/meta-summary のJSONスキーマ拡張
- **自動保存**: タブ非表示時（visibilitychange hidden）に AI要約付きセッション保存を自動実行。3メッセージ以上が条件。`keepalive: true` で信頼性確保。handleEndChat は自動保存済みならAPI呼び出しスキップ。beforeunload では heartbeat（sendBeacon）をフォールバックとして維持

### ビルド・配布
- ビルド確認済み（クライアント）

### 備考
- `chatMessagesRef` パターン: React の useCallback + useState は非同期コールバック内でクロージャ問題を起こす。useRef で最新値を常に参照するのが定石
- Cloud Run min-instances=1 でコールドスタート解消済み
- auto-save の `autoSavedAtRef` でメッセージ数を追跡し、重複保存を防止
