## セッション1: Web対応（Cloud Run デプロイ準備）

### 実装した機能
- **サーバー認証ミドルウェア**: `server/auth-middleware.js` 新規作成。jose で Azure AD JWT 検証（requireAuth + verifyWsToken）
- **クライアント認証付きfetch**: `client/src/lib/fetchWithAuth.ts` 新規作成。MSAL から idToken を取得して Authorization ヘッダーに付加
- **App.tsx 全API呼び出し置換**: 素の `fetch('/api/...')` を `fetchWithAuth` に置換（22箇所）。WebSocket URL にもトークンパラメータ追加
- **server.js 認証適用**: `/api/*` ルートに `requireAuth` ミドルウェア適用（`/healthz` と `/api/ms-subject-token` は除外）。WebSocket 接続時に `verifyWsToken` で検証
- **Dockerfile**: Cloud Run 用。node:18-slim ベース、kuromoji 辞書は npm ci で入る
- **cloud-run-env.yaml**: Azure AD + BQ 設定の環境変数定義
- **deploy-cloud-run.sh**: SA作成・Secret Manager登録・gcloud run deploy のワンストップスクリプト
- **.dockerignore**: 不要ファイル除外

### 備考
- ローカル開発では `AZURE_CLIENT_ID`/`AZURE_TENANT_ID` がサーバー .env に未設定のため認証スキップ（既存 Electron フローに影響なし）
- Cloud Run では ADC（サービスアカウント）で BQ 認証。WIF 不要
- jose パッケージ追加（JWT検証用）
- デプロイ前に Azure AD アプリ登録にCloud Run URLをリダイレクトURIとして追加が必要
